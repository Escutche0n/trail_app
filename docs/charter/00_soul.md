# 产品灵魂锚点 (Soul Anchors)

> **本文件是整个 app 的宪法。所有功能决策必须通过本文件的检验。**
> **修改权限：仅 Elvis 本人在对话中明确要求。AI 不得主动修改本文件。**

---

## 一句话定义

> 一个离线优先的、随着你主动冻结的日子慢慢长出星图的极简时间工具。
> Pro 版把这些日子同步到 Obsidian，好友系统是近场相遇时在自己的星图里埋下朋友星。

---

## 目标用户

- 对隐私敏感的人
- 对打卡游戏化疲劳的人
- Obsidian / 知识管理工具的深度用户
- 会安静凝视自己人生的人
- 对极简美学有要求的人

**不是**：
- 追求 streak 的高频打卡用户
- 追求社交 / 分享 / 比较的用户
- 追求外部成就感和鼓励反馈的用户

---

## 红线（无例外 · 违反即为背叛产品）

### 反游戏化
1. **不做每日上线奖励**。连续天数、签到红点、"别忘了打卡"推送全部禁止。
2. **不显示星星总数 / 打卡总数**。任何 UI 都不出现"你有 X 颗"这种数字。
3. **不做概率掉落 / 稀有度系统**。所有行为 → 结果必须确定性。
4. **不做排行榜 / streak 对比**。

### 反社交异化
5. **不做星星市场 / 交易 / 赠予后消失**。星永远属于它的持有者，赠与 = 共同见证。
6. **不做用户主页 / feed / 公开页面**。
7. **不做 social proof 机制**（"X 位朋友正在打卡"之类）。

### 反数据外泄
8. **不加账号 / 网络同步 / 云端**。数据只在本机 + 用户自己的 Obsidian vault。
9. **不加广告 / 三方追踪 / 分析 SDK**。
10. **不做云端备份**（本地加密导出 + 用户自己管理文件）。

### 反视觉噪声
11. **不做持续背景动画**。动画是标点符号，只在意义发生的瞬间出现。
    **例外 · 星图**：星图上的星本身允许极缓慢的亮度呼吸（周期 ≥ 6 秒、alpha 振幅 ≤ 0.4、每颗星相位随机化、不改位置不改颜色）。理由：星是"内容"，不是装饰；完全静止会被误读为坏点。连线、背景、其他 UI 元素不在此例外范围内。
12. **不加颜色装饰**（星图闪烁除外）。极简黑 + 白为基底。

### 架构红线
13. **生日不可修改**（但"清除所有数据"入口必须存在）。
14. **星图架构**：推导式 + overlay 表。不得改为"星星独立落盘"。
15. **时间轴防篡改必须保持覆盖**。任何新写路径必须通过 `TimeIntegrityService.nowForWrite()`。
16. **Hive 加密盒不得绕过**。任何持久化数据必须通过加密盒。

---

## 气质

- **克制**：默认状态是静。动画、文案、色彩都要通过"是否必要"的检验。
- **诚实**：空白的一周就让它空白，不造假进度，不用鼓励话术。
- **时间感**：任何设计都要能在 5 年后依然成立。不跟随平台最新潮流的组件滥用。
- **重量**：每个交互都应该有一点点分量。默认不允许"一键解决"之类的轻盈承诺。

## 设计判据（当你犹豫时问自己）

1. 这个功能会让用户每天**多打开一次 app**？→ 那就是错的。
2. 这个按钮是在鼓励用户**行动**还是**反思**？→ 只要后者。
3. 如果一个用户一个月没用 app 回来了，这个设计会让他**愧疚**还是**平静地继续**？→ 只要后者。
4. 这个元素在屏幕上**会不会让人看 1 秒就想看第 2 秒**？→ 如果会，它太吵了。
5. 五年后再打开这个 app 的用户，这里**还会成立**吗？→ 如果不会，重新想。
