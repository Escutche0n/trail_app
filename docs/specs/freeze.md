# 主动冻结 (Freeze)

> 状态：设计完成，待 v2.0 实现
> 与 `specs/constellation.md` 绑定

---

## 目标

给用户一个**明确的仪式性动作**来宣告"这一天我认为完成了"。
被冻结的一天会：
1. 不能再修改打卡状态（所有线当天锁定）
2. 在星图上产生一颗确定性的恒定星

---

## 非目标

- 不做"自动冻结"（到点自动锁死 —— 违反主动性）
- 不做"解冻"（一旦冻结永久生效）
- 不做"批量冻结过去的日子"（必须当天或最近一天做，避免补救式使用）

---

## Soul 对齐检查

- ✅ 红线 #1（不做上线奖励）：冻结是主动选择，不是系统赠送
- ✅ 气质"重量"：不可撤销的决定给这个操作分量
- ✅ 气质"诚实"：没冻结的日子就是没冻结，不造假

// 新增字段到 StorageService

迁移：新字段，默认空集合。存量用户无感知。

## 状态机

```
unfrozen (today) ─── [user taps freeze] ───→ frozen (forever)
                                                   │
                                                   ↓
                                          [triggers star birth]
```

规则：
- 只能冻结"今天"或"昨天"（给用户一个晚间缓冲窗口）
- 冻结操作必须通过 `TimeIntegrityService.nowForWrite('freeze')`
- tampered 时拒绝冻结

---

## UI

### 触发入口

**候选位置**（待 Elvis 决策）：
- A: 主界面底部上拉手势区（类似当前 pullUp 基础设施）
- B: 今日节点长按菜单新增一项
- C: 晚间提醒通知里的快捷 action

**倾向：A + B**，C 作为可选增强。

### 冻结动作反馈

1. 用户触发 → 显示一个全屏半透明确认层
2. 文案："冻结 2026 年 4 月 18 日"+"此操作不可撤销"
3. 长按 1 秒（不是 tap，为了仪式感）才真正执行
4. 执行后：触觉反馈（heavy impact）+ 今日节点变为"冻结样式"
5. 同时触发星诞生动画（见 constellation.md）

### 视觉

- 冻结节点的视觉与打卡节点有明确区分
  - 打卡节点：实心白色圆
  - 冻结节点：实心白色圆 + 极细的外环（2px）+ 微弱静态 glow
- **glow 不动画**（违反红线 #11 则 NACK）

### 空状态 / 错误态

- tampered：SnackBar"检测到系统时间异常" + 拒绝
- 已冻结：按钮变灰，无反应
- 非今日非昨日：按钮隐藏

---

## Edge cases

- **跨夜冻结**：用户 23:50 打开确认层，23:59:59 长按完成 → 使用 `nowForWrite` 返回的日期（哪一天就属哪一天，不做"昨天优先"）
- **时区变化（跨时区出行）**：以当前系统时区判断"今天"，但 date key 始终以本地日期存储
- **冻结日的打卡是否锁死**：是。冻结后该天的所有线都不能再 toggle
- **冻结日被后端发现其实没打卡**：保持不打 ✓。"冻结"不等于"完成了所有线"，它只是说"这一天我认了"
